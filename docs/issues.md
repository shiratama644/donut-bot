## 🔴 セキュリティ

### 1. CORS / オリジン検証なし
- **場所**: `src/websocketServer.ts`（`wss.on("connection")`）
- **内容**: WebSocket サーバーはリクエスト元のオリジンを一切検証しない。任意のオリジンから接続して Bot にコマンドを送れる。
- **対処**: `connection` イベントの `request.headers.origin` を検証し、許可オリジン以外は即時切断する。

### 2. チャット入力のバリデーションなし
- **場所**: `src/websocketServer.ts`（40〜50行付近）
- **内容**: 受信メッセージの `text` フィールドに対して文字数制限・内容チェック・レートリミットが存在しない。スパムや大量送信で Bot が不正動作する可能性がある。
- **対処**: 最大文字数の制限、レートリミット（例: 1秒1メッセージ）を実装する。

### 3. WebSocket メッセージのスキーマ検証なし
- **場所**: `src/websocketServer.ts`（`JSON.parse` 後）
- **内容**: 受信した JSON を `Record<string, unknown>` として扱い、型アサーションのみで mineflayer に渡している。不正なペイロードで予期しない動作が起きうる。
- **対処**: `zod` 等のスキーマバリデーションライブラリで入力を検証する。

### 4. `as unknown as BotMessage` による型安全の迂回
- **場所**: `web/hooks/useBotWebSocket.ts`（53行付近）
- **内容**: WebSocket から受け取ったデータを実行時検証なしに型キャストしている。不正なメッセージで実行時エラーが発生する可能性がある。
- **対処**: 判別共用体（discriminated union）とランタイム検証を組み合わせる。

### 5. ログへの機密情報の混入
- **場所**: `src/bot.ts`（`login`, `kicked` イベントハンドラ）
- **内容**: 認証方式・ホスト名・ユーザー名を含むログが Web クライアントに broadcast される。ログが外部に漏れた場合の情報開示リスクがある。
- **対処**: broadcast するログから機密フィールドを除外する。

## 🟠 コード品質

### 6. 空の catch ブロック
- **場所**: `src/websocketServer.ts`（tabcomplete の `.catch()` 部分）、`web/hooks/useBotWebSocket.ts`（`ws.onerror`）
- **内容**: エラーをサイレントに無視してコメントのみで説明している。デバッグが困難になる。
  ```typescript
  .catch(() => {
    // close イベントで再接続
  });
  ```
- **対処**: `catch(err)` として受け取り、少なくともログに出力する。

### 7. モジュールレベルの可変状態 `nextEntryId`
- **場所**: `web/components/ChatPanel.tsx`（101行）
- **内容**: コンポーネント外のモジュールスコープで ID カウンタを管理しており、複数タブを開いた場合に ID が衝突する可能性がある。
- **対処**: `useRef` でコンポーネント内に移動する。

### 8. 型アサーションの不安全な使用
- **場所**: `src/websocketServer.ts`（64行付近）
  ```typescript
  const { text, requestId } = msg as { text: string; requestId: number };
  ```
- **内容**: `typeof` チェック後に型アサーションを重ねているが、実際のプロパティ存在チェックが不足している。
- **対処**: 判別共用体を活用してアサーションを不要にする。

### 9. 使われていない型定義 `ActionbarMessage`
- **場所**: `web/types/bot.ts`（16〜19行）
- **内容**: バックエンドが `actionbar` タイプを broadcast しないため、この型は実質デッドコードになっている。
- **対処**: バックエンドに実装するか、削除する。

### 39. `ChatPanel.tsx` で `e.isComposing` TypeScript ビルドエラー 🆕
- **場所**: `web/components/ChatPanel.tsx`（210行）
- **内容**: React の `KeyboardEvent<HTMLInputElement>` 型に `isComposing` プロパティが存在しないため、Next.js の本番ビルド（`next build`）が TypeScript エラーで失敗する。
  ```typescript
  if (e.key === "Enter" && !e.isComposing) {  // ← TS2339 エラー
  ```
- **対処**: `e.isComposing` を `e.nativeEvent.isComposing` に変更する。

## 🟡 バグ・実行時問題

### 10. `bot.on("move")` リスナーの未解除
- **場所**: `src/bot.ts`（`bot.on("move", ...)` 登録部分）
- **内容**: Bot が切断・再起動された際にイベントリスナーが解除されない。Bot を複数回再起動するとリスナーが累積してメモリリークが発生する。
- **対処**: `end` イベントで `bot.removeListener("move", handler)` を呼ぶ。

### 11. 座標の NaN チェックが X 軸のみ
- **場所**: `src/bot.ts`（34行）、`src/coordinates.ts`（9行）
  ```typescript
  if (!pos || isNaN(pos.x)) return;
  ```
- **内容**: Y・Z 座標が NaN の場合にそのまま broadcast される。
- **対処**: `isNaN(pos.x) || isNaN(pos.y) || isNaN(pos.z)` とすべてチェックする。

### 12. WebSocket の接続先ポートがハードコード
- **場所**: `web/app/page.tsx`（12〜15行）
- **内容**: `NEXT_PUBLIC_WS_URL` 未設定時のフォールバックでホスト名は動的解決されるが、ポート番号は 3000 番固定のため、`WEB_PORT` を変更した本番環境では接続できない。
- **対処**: `WEB_PORT` に合わせてフォールバック URL を動的生成するか、`NEXT_PUBLIC_WS_URL` を必須環境変数にする。

### 13. WebSocket アンマウント後の state 更新
- **場所**: `web/hooks/useBotWebSocket.ts`（40〜45行）
- **内容**: コンポーネントアンマウント時に `wsRef.current?.close()` を呼ぶと `ws.onclose` が非同期で発火し、`setConnected(false)` がアンマウント済みコンポーネントで呼ばれる。
- **対処**: `ws.onclose` / `ws.onopen` 内で `unmountedRef.current` を確認してから state 更新する。

### 14. チャット送信フィードバックなし
- **場所**: `web/components/ChatPanel.tsx`（送信処理部分）
- **内容**: メッセージ送信後に入力欄が即座にクリアされるが、サーバーへの到達確認がなく、失敗しても通知されない。
- **対処**: サーバーエコーの受信後にクリアする、または送信失敗時にエラーを表示する。

### 15. localStorage 操作のエラー区別なし
- **場所**: `web/hooks/useMessageHistory.ts`、`web/hooks/useTheme.ts`
- **内容**: try-catch は存在するが、容量超過エラーとパースエラーを区別しておらず、ユーザーへのフィードバックもない。
- **対処**: エラー種別に応じたフォールバック処理とユーザー通知を追加する。

### 40. ステータス更新間隔が WebSocket 再接続後にリセットされる 🆕
- **場所**: `src/status.ts`、`web/app/page.tsx`、`web/hooks/useBotWebSocket.ts`
- **内容**: 設定パネルでステータス更新間隔を変更した後、WebSocket 接続が切断・再接続されるとサーバー側の間隔が `DEFAULT_INTERVAL_MS`（2000ms）にリセットされる。クライアント UI は最後に設定した値を表示したままのため、表示と実際の動作が乖離する。
- **対処**: WebSocket の `onopen` ハンドラ内で現在の間隔設定を自動的に再送信する。

## 🔵 未実装・不完全な機能

### 16. タブ補完の結果が UI に表示されない
- **場所**: `src/websocketServer.ts`（51〜73行）、`web/components/ChatPanel.tsx`
- **内容**: バックエンドは `tabcomplete` レスポンスを返す実装があるが、フロントエンドはそれを受け取っても補完候補を表示しない。
- **対処**: 入力欄の下に補完候補リストを表示する UI を実装する。

### 17. Actionbar メッセージがバックエンドから送信されない
- **場所**: `src/`（対応する送信処理なし）
- **内容**: フロントエンドの型定義には `ActionbarMessage` があるが、バックエンドは actionbar イベントを broadcast しない。
- **対処**: `bot.on("actionBar", ...)` で受け取り、`ActionbarMessage` を broadcast する。

### 18. 手動再接続ボタンがない
- **場所**: `web/components/ChatPanel.tsx`（接続状態表示部分）
- **内容**: 「再接続中...」メッセージは表示されるが、ユーザーが手動で再接続をトリガーする手段がない。
- **対処**: 切断状態時に「再接続」ボタンを表示する。

### 19. メッセージ検索・フィルタリング機能なし
- **内容**: 最大 500 件のメッセージが保持されるが、検索やフィルタ機能がない。
- **対処**: 検索入力欄とフィルタオプションを追加する。

## 📋 エラーハンドリング不足

### 20. Minecraft 認証エラーのハンドリングなし
- **場所**: `src/bot.ts`
- **内容**: 認証失敗時にプロセスがクラッシュし、再試行ロジックもない。
- **対処**: `error` イベントで認証エラーをキャッチし、適切なメッセージを出力後に再試行または終了する。

### 21. 接続タイムアウトの設定なし
- **場所**: `src/bot.ts`（mineflayer の createBot オプション）
- **内容**: サーバーへの接続が無期限に待ち続ける可能性がある。
- **対処**: `connectTimeout` オプションを設定する。

### 22. グレースフルシャットダウンなし
- **場所**: `src/bot.ts`（`end` イベントで `process.exit()` 呼び出し）
- **内容**: Bot 切断時に `process.exit()` が即座に呼ばれ、WebSocket クライアントのクリーンアップが行われない。
- **対処**: `SIGTERM`/`SIGINT` シグナルハンドラを追加し、接続中クライアントに通知してから終了する。

### 23. ポート競合時のエラーメッセージが不親切
- **場所**: `src/websocketServer.ts`（`server.listen()` 呼び出し）
- **内容**: ポートが使用中の場合、Node.js のデフォルトエラーが表示されるだけで原因が分かりにくい。
- **対処**: `error` イベントで `EADDRINUSE` を検出し、分かりやすいメッセージを出力する。

## ⚙️ 設定・デプロイ

### 28. `.env.example` のバージョンへのコメントが不足

- **対処**: 起動時に必須変数をチェックし、不足していれば明確なエラーメッセージで終了する。

## ♿ アクセシビリティ

### 33. チャットエリアに ARIA ロールなし
- **場所**: `web/components/ChatPanel.tsx`（`.chat-messages` 要素）
- **内容**: チャットログ要素に `role="log"` や `aria-live="polite"` が設定されておらず、スクリーンリーダーが新着メッセージを通知できない。
- **対処**: チャットコンテナに `role="log"` と `aria-live="polite"` を追加する。

## ⚡ パフォーマンス

### 34. 座標 broadcast のデバウンスなし
- **場所**: `src/coordinates.ts`
- **内容**: `COORD_DISPLAY_INTERVAL_MS` 間隔で無条件に座標を broadcast しており、クライアント数が増えると不必要な通信が発生する。`bot.on("move")` には `MOVE_THROTTLE_MS` によるスロットリングがあるが、インターバル側には変化検出がない。
- **対処**: 前回と座標が変化した時のみ送信するか、接続中クライアントが存在する場合のみ送信する。

### 35. メッセージリストの仮想化なし
- **場所**: `web/components/ChatPanel.tsx`
- **内容**: 最大 500 件のメッセージが全件 DOM にレンダリングされる。大量メッセージ時に UI がラグを起こす可能性がある。
- **対処**: `react-window` 等で仮想スクロールを実装する。

## 🔧 インフラ・運用

### 36. ヘルスチェックエンドポイントなし
- **場所**: `src/websocketServer.ts`
- **内容**: WebSocket サーバーは基本的な HTTP GET を返すが、ヘルスチェック専用のエンドポイントがなくモニタリングツールとの統合が困難。
- **対処**: `/health` エンドポイントを追加し、Bot の接続状態や WebSocket クライアント数を返す。

### 37. WebSocket へのレートリミットなし
- **場所**: `src/websocketServer.ts`
- **内容**: 1 クライアントから秒間無制限のメッセージを送信でき、DoS 攻撃の対象になりうる。
- **対処**: 接続ごとのメッセージレートリミットを実装する。

### 38. `dist/` の古いファイルのクリーンアップなし
- **場所**: `package.json`（`scripts`）
- **内容**: TypeScript のビルド出力 `dist/` に古いファイルが残ったまま蓄積される可能性がある。
- **対処**: `"clean": "rm -rf dist"` スクリプトを追加し、`build` スクリプトで `clean` を先行実行するようにする。
